邮件发送提示错误信息(400,500)大全
分类:营销策略 | 评论:0 | 浏览:


今天公司有同事反映发送邮件错误，看了下系统退信的错误提示是：550 #5.1.0 Address rejected.仅从字面意思来看，是地址被拒绝，去网上搜下一下，发现有这个邮件错误问题的帖子，问题比较全，赶紧转过来，这样以后遇到类似的问题可以对照查看解决了。

邮件服务器错误报告

400 错误报告

Q:421 Server too busy
A:因为收件方服务器问题无法连接,这是收件方服务器的SMTP session Full或者是因为收件方服务器 Traffic 无法进行新的 SMTP 连接。请过一会儿重新发送，如继续发生错误时向收件方管理员咨询。

Q:421-Microsoft ESMTP MAIL Service, Version: 5.0.2195.5600 ready at Service not available, closing transmission channel
A:收件方的 Microsoft ESMTP 服务器无法正常工作引起的发送失败.请过一会儿重新发送，如继续发生错误时向收件方管理员咨询。

Q:421 4.3.1 ILLEGAL FROM <xxx@xxx.com> Sorry. Your IP has problems. This Connection will be closed. Please mail to mailsysop@hanmir.com (by using another e-mail server). Thank you.
A:由于收件方IP地址错误引起的发送限制错误。向显示的咨询邮箱地址发送邮件咨询。

Q:421 4.3.2 Your IP(xxx.xxx.xxx.xxx) is filtered and this connection will be closed. You must register your IP to spammaster@hanafos.com By SpaceLee, the Lord of mail server
A:发送人的IP地址在 www.hanafos.com 的网址上登录为SPAM邮件引起的发送限制错误。向显示的咨询邮箱地址发送邮件咨询。

Q:421 4.3.2 Your IP(xxx.xxx.xxx.xxx) is filtered and this connection will be closed. You must register your IP to http://realip.naver.com.
A:发送人的IP地址在 www.naver.com 的网址上登录为SPAM邮件引起的发送限制错误。请参考naver.com的spam政策。

Q:441 4.4.1 No answer from host
A:收件方邮件服务器没有应答失败的连接。请过一会儿重新发送，如继续发生错误时向收件方管理员咨询。

Q:441 4.4.1 temporay dns failed <Hanmail.net>
A:连接hanmail.net的时候因违反hanmail.net的spam政策引起的错误。请联系 1544-0580 分机 2 咨 询

Q:450 <xxx@xxx.xxx> : Recipient address rejected: Greylisted for 300 seconds (see http://isg.ee.ethz.ch/tools/postgrey/help)
A:发送方的邮件地址在收件方的 Graylisting 系统上已登陆并存在引起邮件发送延长的错误。

Q:451 4.4.0 DNS resolving error
A:没有找到收件方的网址出现的错误。请确认现在邮件服务器拥有的DNS服务器是否正常运行。可以试一下参考其他ISP企业的DNS发送邮件。

Q:451 4.3.0 Other or undefined mail system status
A:收件方邮件系统PROTOCOL错误或连接的服务器并不是SMTP时发生的错误。

Q: 451 4.3.0 Temporary system failure. Please try again later
A:因收件方服务器的障碍无法收件的错误。

Q:451 4.4.2 Bad connection (io timeout)
A:因收件方服务器应答时间超过断开连接发生的错误。请过一会儿重新发送，如继续发生错误时向收件方管理员咨询。

Q:451 Relay Server Not Ready
A:收件方服务器的一系列功能没有正常工作引起的错误。请过一会儿重新发送，如继续发生错误时向收件方管理员咨询。

Q:452 sorry, your recipient address is not in my local list
A:在收件方的邮件服务器里不存在所发送的地址。请确认收件人的邮件地址是否正确并且确认此地址是否在收件方实际存在的地址。

Q: 452 4.4.5 Requested mail action aborted: exceeded storage allocation
A:收件方邮件服务器储存空间不够引起的错误。请向收件方管理员咨询。

Q:452 4.4.5 Disk write / Insufficient disk space error occurred
A:收件方邮件服务器储存空间不够引起的错误。请向收件方管理员咨询。

Q: 452 4.4.5 Insufficient disk space; try again later
A:收件方邮件服务器储存空间不够引起的错误。请向收件方管理员咨询。

Q:452 4.4.5 MailBox or MessageQuota exceed for <E-mail Address>
A:收件人的个人邮箱空间不够引起的错误。请向收件方管理员咨询。

500 错误报告

Q:5XX you must be spam
A:发送方的邮件服务器网址或IP地址及邮件地址被收件方邮件服务器登陆为垃圾清单引起的错误。向收件方管理员咨询请求从垃圾清单上删除。

Q:5XX Your envelope sender is enlisted as spammer
A:发送方的邮件地址被收件方的邮件服务器判定为垃圾邮件地址而引起的错误。向收件方管理员咨询请求从垃圾清单上删除。

Q:500 Syntax error, command unrecognized
A:SMTP 命令语中一些被漏掉或一些错误的命令语引起的错误。

Q: 500 File Sending Error
A:传送邮件时因网络的障碍等原因而导致传送失败。

Q:501 Denied domain name
A:发送方的网址被收件方的服务器拒绝而引起的错误。

Q:501 Bad sender address expression
A:发送人或收件人的邮箱地址错误引起的。请确认发送人的邮件地址是否正确。

Q: 501 Argument missing
A:SMTP 命令语中一些被漏掉或一些错误的命令语引起的错误。

Q:501 5.1.8 Sender domain must exist <xxx.com>
A:收件方的 xxx.com 网址不存在而引起的错误。请确认收件人邮件地址中的网址是否正确。如地址正确请向收件方管理员咨询是否已登陆在实际网址相关的邮件服务器上。

Q:503 Bad sequence of commands
A:SMTP 命令语中一些被漏掉或一些错误的命令语引起的错误。

Q:504 You are registered as spammer
A:发送方的邮件服务器网址或IP地址及邮件地址被收件方邮件服务器判定为垃圾清单内的邮件地址而引起的错误。向收件方管理员咨询请求从垃圾清单上删除。

Q:504 We do not accept mail from spammers. if you have questions,please Email
A:发送方的邮件服务器网址或IP地址及邮件地址被收件方邮件服务器判定为垃圾清单内地址而引起的错误。向收件方管理员咨询请求从垃圾清单上删除。

Q:511 sorry, no mailbox here by that name (#5.1.1 - chkuser)
A:收件方的邮件地址在收件方的邮件服务器上不存在引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 No inbox for <E-mail Address>
A:收件方的邮件地址在收件方的邮件服务器上不存在或停止使用而引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 User unknown
A:收件方的邮件地址在收件方的邮件服务器上不存在或停止使用引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 Requested action not taken: mailbox unavailable
A:收件方的邮件地址在收件方的邮件服务器上不存在或停止使用引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 5.1.1 Suspended mailbox
A:收件方的邮件地址在收件方的邮件服务器上不存在或停止使用引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 5.1.1 No such user <E-mail Address>
A:收件方的邮件地址在收件方的邮件服务器上不存在或停止使用引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 Invalid recipient
A:收件方的邮件地址在收件方的邮件服务器上不存在或停止使用引起的错误。请确认收件人的地址是否正确。收件人的账号是否为可使用的账号，并向收件方管理员咨询。

Q:550 5.0.0 Access denied
A:被收件方邮件服务器拒绝登陆而引起的错误,向收件方邮件服务器确认为何发送方的邮件服务器被拒绝。

Q:553 sorry, that domain isn't in my list of allowed rcpt hosts
A:收件方的网址在收件方的邮件服务器上不存在引起的错误。请确认收件人的邮件地址是否正确，确认收件方的网址在收件方邮件服务器上是否存在。

Q:550 Relay rejected for <E-Mail Address>
A:在收件方邮件服务器拒绝相关邮件地址一系列发生的错误。向收件方管理员咨询。

Q:550 relaying to <E-Mail Address> prohibited by administrator
A:在收件方邮件服务器拒绝相关邮件地址一系列发生的错误。向收件方管理员咨询。

Q:553 Too many recipients
A:收件方邮件服务器允许的同时超过收件总数发生的问题。请根据收件方邮件服务器允许的数量发送邮件。

Q:553 sorry, your envelope sender is in my badmailfrom list.
A:发送方的邮件服务器网址或IP地址及邮件地址被收件方邮件服务器判定为垃圾清单内的地址而引起的错误。向收件方管理员咨询请求从垃圾清单上删除。

Q:553 sorry, your envelope sender is enlisted as spammer
A:发送方的邮件服务器网址或IP地址及邮件地址被收件方邮件服务器判定为垃圾清单内的地址而引起的错误。向收件方管理员咨询请求从垃圾清单上删除。

Q:550 5.0.0 Your message may contain the Klez.worm! See http://securityresponse.symantec.com/avcenter/venc/data/w32.klez.removal.tool.html
A:发送的邮件里含有 Klez worm 病毒，因此收件方邮件服务器拒绝。请检验电脑及服务器的状态。如有附件请确认附件是否带有病毒后再次发送。

Q:553 5.0.0 Your message may contain the Win32.Klez worm!!- If you have questions,please email
A:发送的邮件里含有 Win32.Klez worm病毒因此收件方邮件服务器拒绝。请检验电脑及服务器的状态。如有附件请确认附件是否带有病毒后再次发送。

Q:553 5.5.4 Host name (xxx.com) is not match with your ip(xxx.xxx.xxx.xxx), maybe it's bogus. Visit http://antispam.hanmail.net
A:收件方的邮件服务器对发送网址 xxx.com做DNS Lookup，但与发送方的IP地址 xxx.xxx.xxx.xxx 不一致拒绝发送的错误。形成对收件方网址HOST（A）记录后将其与发送邮件服务器IP地址变为一致。

Q:553 5.5.4 <xxx> ... Domain name required for sender address <xxx>
A:发送方的邮件地址并非是 ID@Domain_Name 形式地址发送引起的错误。请确认发送人的邮件地址是否正确。

Q:554 5.3.4 Too big mail sent. Max allowed mail size is xxxxxxxxx bytes
A:比收件方邮件服务器允许的邮件容量大引起的发送错误。请检查发送邮件大小并根据允许邮件容量再次发送。

Q:554 5.3.0 Mail have traversed Too many hops. Reject it
A:收件方邮件服务器允许的同时超过收件总数发生的问题。请根据收件方邮件服务器允许的数量发送邮件

Q:554 delivery error: dd Sorry, your message to <E-Mail Address> cannot be delivered. This account is over quota
A:收件人的个人邮箱空间不够引起的错误。请向收件方管理员咨询。

Q:554 5.1.0 Sender Denied
A: 发送方的邮件地址被拒绝发生的错误。请向收件方管理员咨询。

Q:554 5.5.0 Your message was rejected. Please refer to http://antispam.nate.com/554-550.html [Reject Option]
A:违反NATE.COM垃圾政策引起的无法发送邮件错误。请确认NATE.COM的垃圾政策。

Q:554 delivery error: dd This user doesn't have a yahoo.com account (E-Mail Address) [-5] - mta114.mail.sc5.yahoo.com
A:收件人的邮件地址实际上在yahoo.com上不存在引起的问题。请确认收件人的邮件地址是否正确输入。

Q:550 5.7.1 <E-Mail Address>. Relaying denied. IP name lookup failed <IP Address>
A:发送人的邮件地址在 DNS IP Lookup 上失败被拒绝一系列服务。

Q:550 5.5.0 VIRUS detected from your mail. Cure & Try again. Found by virus scan engine.
A:发送的邮件里含有病毒被收件方拒绝。请检验电脑及服务器的状态。如有附件请确认附件是否带有病毒后再次发送。

Q:550 Denied by policy: Sender is listed on DNS-based RBL
A:RBL (Real time Black List) 上含有发送方的邮件服务器网址或IP地址及邮件地址因此收件方邮件服务器拒绝。请确认登陆发送方网址或IP地址的RBL从清单上删除。从清单上删除的方法每个RBL网站都不同，请根据相应网站指示操作。

Q:550 Mail sender unuses allowed IP. SMTP Proxy Server close ...
A:发送方IP地址被收件方邮件服务器拒绝发生错误。向收件方管理员咨询是否发送方的IP地址被拒绝后再删除。

Q:553 5.5.3 <E-Mail Address>... Invalid
A:发送人或收件人的邮件地址错误而发生问题。请确认发送人或收件人的邮件地址是否正确。

Q:557 you must be virus
A:发送的邮件里含有病毒被收件方拒绝。请检验电脑及服务器的状态。如有附件请确认附件是否带有病毒后再次发送。

Outlook 错误报告

Q:Login 到邮件服务器发生问题。指定的密码被拒绝。账号:‘pop.mail.yahoo.com’, 服务器: ‘pop.mail.yahoo.co.kr’, Protocol: POP3, 服务器应答: ‘-ERR Bad login’, port 110, 安全(SSL): 否, 服务器错误: 0x800CCC90, 错误编号: 0x800CCC92►
A:在Outlook[工具]->[账号]->[登陆信息]->[服务器]->密码变更或在 SiteControl LOGIN后 通过[账号管理员] ->[密码变更]可重新指定密码。但也有时候邮件账号使用量超过时也会发生这种情况。

Q: 无法找到‘mail.enagex.com’ 的host.请确认是否正确的输入服务器名。账号: ‘admin’, 服务器: ‘mail.enagex.com’, Protocol: POP3, port: 110, 安全(SSL): 否, socket错误: 11001, 错误编号: 0x800CCC0D从服务器上无法连接。账号: 'mail.xxxx.xx.'
A:重新正确的确认 POP3的名称后修改 POP3服务器名称。 例>ENAGE POP3 : mail.enage.com

Q:[无法连接到服务器. 账号: ‘mail.xxxx.xx.xx’, 服务器: ‘127.0.0.1’, Protocol: POP3, port: 1110, 安全(SSL): 否, socket错误: 10061, 错误编号: 0x800CCC0E]
A:设置V3Pro 2002 Deluxe后 [POP3监视], [Outlook 监视]打开就是[On]时 Outlook Express的账号部分会像 127.0.0.1一样外加 Local的 IP。
收件服务器 : 127.0.0.1
账号名称 :账号名称/原邮件服务器地址/110

[127.0.0.1][proxy server］会在用户登陆服务器查看邮件之前检验病毒.
如果账号变更后 POP3 监视发生错误装置为如上状态上 POP3监视模块无法正常工作那么也无法收件.这种情况下可以用以下方法调整.
1. 在V3Pro 2002 Deluxe的实施监视上关掉POP3 的监视, Outlook监视 .
2. 关掉V3和 Outlook Express后重新启动 Outlook Express [工具]-[账号]-[邮件]上选择设置POP3监视的账号.
在[登陆信息]->[高级]上受服务器端口设置的邮件换成110后, 在V3Pro 2002 Deluxe上 POP3 监视和 Outlook 监视从新设置为 On既可正常收件了.

Q:服务器连接突然中断、服务器问题、 网络问题或连接后很长时间没有使用的原因也会存在.
账号 : ‘xxxxxxx’, 服务器 :‘mail.xxxxx.xxx’, protocol: POP3, 服务器应答: ‘+OK 4096 octers’, 端口: 110, 安全(SSL):否, 错误编号: 0x800CCC0F
A:电脑上安装杀毒软件或硬件时发生问题。杀毒软件或硬件在服务器检测上下载的邮件，邮件中存在被认为病毒时为电脑的安全中断与服务器的连接。
所以这时候解决方法就是用电子邮件登陆后被认为感染病毒的文件删除再重新下载既可。
(也就是删除处理邮件中的广告性邮件或未确认的发送人发送过来的邮件。)

Q:‘mail.enage.com’ 无法找到host. 请确认服务器名是否正确输入。 账号: ‘orbiter’, 服务器: ‘mail.enage.com’, protocol: SMTP, 端口: 25, 安全(SSL): 否, socket错误: 11001, 错误编号: 0x800CCC0D
A:请从新确认SMTP服务器名称后修改。
SMTP 服务器名称如下
SMTP/POP3 : [gw.顾客域名] or [mail.顾客域名]

Q:‘mail.enage.com’ 无法找到host。请确认是否正确的输入服务器名。账号: ‘orbiter’, 服务器: ‘mail30.enage.com’, protocol: SMTP, 端口: 25,安全(SSL): 否, socke错误: 11004, 错误编号: 0x800CCC0D
A:确认收到的信(POP3)是否正确。

(※ 如使用旧版本的V3杀毒软件时端口的求和如上转换为 1110。删除 V3后从新设置邮件账号。)

Q: 收件人中有一个人被服务器拒绝无法传送信息。被拒绝的电子邮件地址是‘bulsae76@yahoo.com’ 题目 ‘test’, 账号: ‘bulsae76’, 服务器: ‘yahoo.com’, protocol: SMTP, 服务器应答: ‘553 RCPT
A:这时需要联系当前使用的网络宽带提供商，后根据此企业SMTP服务器设置相应的SMTP才可发送邮件.因每个通信公司只限制给自己的会员使用 SMTP因此不能通过其它通信公司发送邮件.

Q:使用安全密码认证无法在服务器上登录.
账号: ‘abc.co.kr’, 服务器: ‘abc.co.kr’, portocol: POP3, 服务器应答: ‘.’, 端口: 110,
安全(SSL): 否, 错误编号: 0x800CCC18
A:->[工具] - [账号] - [邮件] -> 装置邮件的 [登陆信息] -> [服务器] 部分上
-> ‘使用安全密码认证(SPA)登 陆 ‘ 解除确认
